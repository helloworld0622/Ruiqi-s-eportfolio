Network Intrusion Detection Module for DoS Attacks

Conducted DoS attacks using Kali Linux and captured network traffic with Wireshark. Extracted features with CICFlowMeter.Implemented and evaluated a Random Forest-based intrusion detection system for detecting Denial of Service (DoS) attacks.

GitHub

Project Overview

In this project, I developed a Random Forest classifier to detect anomalies in network traffic, specifically distinguishing between benign and Denial-of-Service (DoS) attacks. The model was constructed with 50 decision trees, each with a maximum depth of 10, and required at least 4 samples to split an internal node. We trained and tested the model on a dataset split into 80% training and 20% testing data. To evaluate the model, we simulated DoS attacks alongside normal activities using Kali Linux, captured the network traffic with Wireshark, and labeled the data based on destination ports. The trained model was then used to classify each record, and its predictions were compared against the actual labels.

The results showed a high performance of the Random Forest classifier, with the ROC curve indicating a high true positive rate and a low false positive rate, and an AUC value of 0.88. The model achieved an overall accuracy of 85%, with nearly perfect precision, recall, and F1-score for both classes. For DoS traffic, the precision was 1.00, meaning no false positives, and the recall was 0.76, indicating some false negatives. For benign traffic, the precision was 0.73, with a perfect recall of 1.00, indicating no false negatives but some false positives. At the best operating point, the false positive rate was 0.00, and the detection rate was 0.76, showing that the model accurately identified all negative instances and 76% of the positive instances. This demonstrates the model's effectiveness in distinguishing between benign and DoS network traffic.
